## 产品目标
- 让用户在 **10 分钟内** 完成一份可导出、可复用、可继续编辑的在线简历。
- 用 **结构化编辑 + 实时预览** 替代纯文档式编辑，降低排版成本。
- MVP 只验证三件事：**能登录、能管理多份简历、能高效编辑并导出 PDF**。

> **MVP判断标准：** 用户首次进入后，可完成注册/登录、新建简历、编辑内容、切换模板、调整排版、导入导出 JSON、导出无水印 PDF，全流程无人工介入。

---

## 范围边界
**本期包含：**
- 邮箱密码注册与登录
- JWT 鉴权
- 简历列表与多份简历 CRUD
- 结构化简历编辑
- 实时预览
- 模板切换
- 基础排版控制
- JSON 导入导出
- 无水印 PDF 导出

**本期不包含：**
- 多人协作
- AI 生成/润色
- 在线分享页
- 简历投递追踪
- 国际化
- 复杂权限体系
- 富文本自由拖拽排版

---

## 用户与使用场景
**目标用户：** 需要快速生成、维护、导出简历的求职者。

**核心场景：**
- 首次注册后快速创建第一份简历
- 为不同岗位维护多份简历版本
- 基于模板快速切换呈现风格
- 导入既有 JSON 数据继续编辑
- 导出正式 PDF 用于投递

## 页面一：产品落地页（Home Page）

### Layout
- 顶部 Header：左侧产品名 / Logo；右侧根据登录态显示不同操作。
  - 未登录态：显示“注册 / 登录”按钮。
  - 已登录态：显示用户头像，Hover 展开下拉菜单，包含“控制台”“退出登录”等入口。
- Hero Section：展示产品名称、核心价值主张与主 CTA `Get Started`。
- 登录 / 注册通过 Modal 承接，不占首页主版面。
- 底部 Footer & Features：展示核心特性、操作引导与补充跳转入口。

### Functions
- **Header**
  - 根据 JWT 登录态切换“注册 / 登录”按钮或头像菜单。
  - 头像菜单提供“控制台”“退出登录”等操作。
- **Hero Section**
  - 展示产品名称与核心价值主张。
  - 提供醒目的 `Get Started` 按钮。
- **登录 / 注册 Modal**
  - 支持登录 / 注册 Tab 切换。
  - 注册字段：邮箱、密码、确认密码；登录字段：邮箱、密码。
  - 提交成功后签发 JWT，前端持有并用于后续受保护接口认证。
- **Footer & Features**
  - 展示核心特性：完全免费、无水印 PDF、JSON 导入 / 导出、多模板与排版调整。
  - 提供功能说明、进入控制台等跳转入口。

### Interactions
- 用户首次进入默认浏览产品落地页，无需登录。
- 未登录用户点击 `Get Started` 或 Header 中“注册 / 登录”按钮时，弹出登录 / 注册 Modal。
- 已登录用户点击 `Get Started`，或从头像下拉菜单点击“控制台”时，直接跳转工作台首页。
- Modal 内支持登录 / 注册切换；注册时即时校验邮箱格式与两次密码一致性。
- 点击“退出登录”后清除 JWT 并回到首页未登录态；JWT 失效后再次触发受保护操作时重新拉起 Modal。

## 页面二：控制台 / 工作台首页

### Layout
- 顶部导航：产品名、用户信息、退出登录。
- 主内容区：简历列表。
- 列表上方：新建简历按钮、搜索框（可选弱化为名称筛选）。
- 列表项：简历标题、最近更新时间、模板名称、操作区。
- 空状态：引导创建第一份简历。

### Functions
- **简历列表展示**
  - 查看当前账号下所有简历
  - 显示标题、更新时间、模板信息
- **CRUD**
  - 新建简历
  - 打开简历
  - 重命名简历
  - 删除简历
  - 复制简历
- **基础管理**
  - 按标题搜索/筛选
  - 按最近更新时间排序

### Interactions
- 点击“新建简历”后创建空白简历并进入编辑工作区。
- 点击列表项或“编辑”进入对应简历。
- 删除前二次确认，避免误删。
- 复制后生成一份新简历，默认名称为“原名称 - 副本”。
- 空状态下主按钮唯一聚焦“创建第一份简历”。

## 页面三：核心编辑工作区

### Layout
- 双栏主布局。
- **左侧：表单与结构化数据管理区**
  - 分区折叠面板，如个人信息、教育经历、工作经历、项目经历、技能、奖项、自定义模块。
  - 当前模块字段表单。
  - 区块排序与增删入口。
- **右侧：实时预览区**
  - 按当前模板实时渲染简历。
  - 固定预览画布，支持滚动查看多页。
- 顶部工具栏：返回工作台、简历标题、保存状态、模板切换、排版控制、JSON 导入导出、PDF 导出。

### Functions
#### 1. 结构化数据管理
- 按模块录入简历字段。
- 支持模块新增、编辑、删除。
- 支持经历类条目多条录入。
- 支持模块排序、条目排序。
- 支持显示/隐藏某个模块。
- 支持有限的自定义模块：模块名 + 文本内容。

#### 2. 表单编辑
- 个人信息：姓名、职位、联系方式、个人摘要。
- 教育经历：学校、专业、学历、时间、描述。
- 工作经历：公司、岗位、时间、职责描述。
- 项目经历：项目名、角色、时间、项目说明。
- 技能/奖项等：标题 + 内容。
- 所有字段以结构化方式存储，避免自由排版带来的不可控性。

#### 3. 实时预览
- 左侧改动实时反映到右侧预览。
- 预览与导出结果保持高度一致。
- 多页内容自动分页。

#### 4. 模板切换
- MVP 提供 2 到 3 套模板。
- 同一份结构化数据可一键切换模板。
- 模板只影响展示样式，不改变原始数据。

#### 5. 排版控制
- 字体大小
- 行距
- 页边距
- 主色/强调色
- 模块间距
- 是否显示头像（如产品支持头像字段，可作为可选开关）

#### 6. JSON 导入导出
- 导出当前简历 JSON
- 导入 JSON 覆盖当前简历或创建新简历
- 对 JSON 基本结构合法性做校验

#### 7. 无水印 PDF 导出
- 将当前预览导出为 PDF
- 导出文件无平台水印
- 保证常见 A4 简历版式稳定

#### 8. 数据保存
- 自动保存
- 关键操作后即时保存
- 保存状态可见，如“已保存”“保存中”“保存失败”

### Interactions
#### 1. 编辑交互
- 左侧字段失焦或短延时后自动保存。
- 经历类模块支持“新增一条”“删除”“拖拽或按钮排序”。
- 删除条目前需确认；简单字段清空不额外确认。
- 模块折叠后保留已填内容，不影响预览展示。

#### 2. 预览交互
- 右侧预览随编辑实时刷新，不要求用户手动点击“预览”。
- 模板切换后立即刷新预览。
- 排版参数修改后立即作用于当前预览。

#### 3. JSON 交互
- 点击导出后直接下载 `.json` 文件。
- 点击导入后选择本地 JSON 文件，校验通过才允许写入。
- 校验失败时明确提示字段结构错误，不写入脏数据。

#### 4. PDF 交互
- 点击导出 PDF 后进入处理中状态。
- 导出成功后自动下载或提供下载入口。
- 若内容溢出导致分页异常，应给出基础提示，但优先保证文件可导出。

#### 5. 异常与恢复
- 自动保存失败时保留本地编辑状态，并提示重试。
- JWT 失效时阻止继续提交，提示重新登录。
- 导入失败、导出失败均给出明确原因，不使用模糊提示。

## 关键流程
### 流程一：首次使用
1. 用户注册并登录。
2. 进入工作台。
3. 新建第一份简历。
4. 在编辑区填写内容。
5. 切换模板并微调排版。
6. 导出 PDF。

### 流程二：维护多份简历
1. 用户在工作台复制已有简历。
2. 修改标题与部分经历内容。
3. 保存为岗位定制版。
4. 单独导出该版本 PDF。

### 流程三：外部数据迁移
1. 用户进入编辑页。
2. 导入本地 JSON。
3. 系统校验并写入结构化数据。
4. 用户继续编辑并导出 PDF。

## 关键规则
- 一个账号可创建多份简历。
- 简历内容以 `contentJson` 结构化存储。
- 模板与内容解耦，模板切换不改写原始数据。
- 导出的 PDF 默认无水印。
- 未登录用户不可访问工作台与编辑页。

## MVP 验收标准
### 功能验收
- 用户可用邮箱密码完成注册与登录。
- 登录后可获得并使用 JWT 访问受保护接口。
- 用户可在工作台完成多份简历的新增、查看、编辑、复制、删除。
- 用户可在编辑区完成核心模块的结构化编辑。
- 编辑结果可实时预览。
- 至少支持 2 套可用模板切换。
- 至少支持 4 项排版控制能力：字体大小、行距、页边距、主题色。
- 用户可成功导入、导出 JSON。
- 用户可成功导出无水印 PDF。

### 体验验收
- 首次创建并导出一份简历，全流程不超过 10 分钟。
- 常规编辑操作无需手动保存即可稳定持久化。
- 页面结构清晰，核心操作 1 次点击可达。
- 失败提示可理解、可恢复。

### 技术验收
- JWT 鉴权链路完整可用。
- 简历数据按用户隔离。
- 导出 PDF 结果与预览视觉基本一致。
- 关键页面在桌面端主流浏览器可正常使用。

## 版本结论
该 MVP 只解决一件事：**让用户高效产出多版本、可导出、版式稳定的在线简历**。若上述核心链路成立，再进入下一阶段评估分享、协作、AI 能力与开放社区建设。
